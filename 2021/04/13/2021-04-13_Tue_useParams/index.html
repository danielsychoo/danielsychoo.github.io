react-router-dom의 useParams() 이용 (2021-04-13 화)

useParams

params를 이용해 route를 정할 때

route를 설정하는 과정 중 params를 이용하게 될 경우 바로 params를 이용한 route를 지정하기보다 앞에 movie-details와 같은 내용을 함께 넣는게 좋음.

만일 이렇게 하지 않은채 params만으로 route를 설정하면 어떤 값을 넣어도 route에 걸려 404 not found 페이지를 분기할 수 없기 때문.

이를 고려하여 route를 설정한다면 아래와 같은 예시가 될 것임.

위 처럼 route를 작성한다면, user가 지정한 route외에 다른 경로로 접속했을 때 모두 404notFound 페이지로 이동될 것임.

404 not found 페이지

위의 상황을 반대로 표현한다면, 이는 user가 정의되지 않은 route를 요청했을 경우 404 not found 페이지로 이동되어야 한다는 것이다. 즉, 404 페이지가 구축되어 있어야 한다는 것.

어떻게보면 너무 당연한 얘기인데, 나는 지금까지 자동으로 404 페이지로 연결된다고 생각했었다. 만일 404 페이지를 구축하지 않았다면, user는 아무것도 없는 공백 페이지를 마주하게 된다. 이는 UX측면에서도 나쁜 상황. (없으면 없다고라도 띄워줘야 User가 상황을 인지함)

useParams()

params를 이용해 route를 분기했을 경우, useParams()를 이용해 해당 값을 가져올 수 있음. 이 것이 무슨 의미인지 아래 실예를 들어보자면,

상황: home에서 특정 content를 클릭 시 해당 content의 detail page로 연결

해당 상황에 조치를 취하려면 user가 클릭한 특정 content의 Id를 detail page로 보내줘야 함. component tree는 다음과 같음.

  • App.js
    • Home.js
      • SingleMovie.js
    • DetailPage.js

1. 부모 component의 함수로 추출 (지금까지의 방법)

  1. 부모 component에서 필요한 값을 setState하는 함수를 만들어
  2. 자녀 component로 내려주고
  3. 필요 값을 함수의 parameter로 담아 가져옴
  4. 이후 해당 값을 필요 component에 props로 전달

매우 번거롭고 문제발생의 가능성이 많음.

2. useParams()를 이용 (새로 알게 된 방법)

  1. import { useParams } from ‘react-router-dom’;
  2. const data = useParams();

Apollo-client query의 key와 type

Apollo-client에서 query를 날릴 때 해당 query에 변수가 있을 경우 변수의 key를 고려할 것.

위 코드를 보면 id로 key를 정해놓고 그 data를 담아놓은 변수이름인 ClickedId를 넣고있음. 이 경우
variables: { id: ClickedId }와 같은 형식으로 넣어야 함.

또한 해당 query의 value type을 Int로 지정해놓았으므로, 통신을 통해 value가 String으로 온다면 parsing해줘야 한다는 점을 잊으면 안됨.

  • String을 Int로 바꿀 때 (parseInt(변수))
  • String을 Float으로 바꿀 때 (parseFloat(변수))
  • String을 number로 바꿀 때 (String(변수))
