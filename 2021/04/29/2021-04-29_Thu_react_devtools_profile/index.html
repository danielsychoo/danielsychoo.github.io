react devtool을 통한 성능확인 (2021-04-29 목)

react 성능

react로 만들어진 웹의 성능은 리렌더링시에 불필요한 정보 및 컴포넌트의 리렌더를 얼마나 줄이느냐가 관건이다. 이 부분은 Chrome extention중 React DevTools의 profiler를 사용하여 확인 및 테스트할 수 있다.

Profiler

React Devtools중 Components 탭은 컴포넌트 사이에서 props가 잘 전달되고 있는지, 혹은 액션에 따라 state가 잘 관리되고 있는지 등을 확인하기 위해 앱을 구현하는 과정에서 자주 확인하게 되는데, profiler 탭의 경우에는 구현이 된 이후 성능 및 속도등의 부분을 확인하기 위해 사용된다.

사실상 profiler는 앱의 활동과정을 녹화한 후 그 녹화 중에 내부에서 벌어지는 렌더링을 잡아내고 시간이 얼마나 걸리는지 확인해주는 방식으로 동작한다. 따라서 녹화버튼을 누른 후 테스트할 페이지를 수동으로 들어가야 한다.

  1. profiler 탭의 녹화버튼 누르고
  2. 테스트할 페이지 렌더링 실시
  3. 다시 녹화버튼 눌러 녹화를 끄고
  4. 결과 확인 및 분석

profiler 결과창 분석

profiler의 결과창에서 우리가 파악할 수 있는 부분은 아래 그림과 같다.

먼저, 회색 으로 나타나는 컴포넌트는 녹화과정에서 렌더링 된 적이 없는 부분이다. 따라서 리렌더링을 원치 않는 부분이 회색으로 나타나지 않는다면 조치가 필요하다.

Router의 경우 Router가 설정되어있는 컴포넌트가 첫 렌더될 시에만 렌더링 되고, 이후에는 늘 리렌더링이 되지 않는다.

반대로 색상이 들어간 부분은 렌더링이 일어난 부분이다. 해당 그림중 Navigation 컴포넌트에 보면 0.3ms of 1.5ms라는 부분이 있는데, 1.5ms라는 이 페이지의 전체 렌더링 시간 중 Navigation 컴포넌트의 렌더링 시간이 0.3ms만큼 잡아먹었다는 뜻이다.

이를 통해 내부의 컴포넌트들 중 어느 컴포넌트의 성능이 떨어지는지를 확인할 수도 있다. 확인 후 해당 컴포넌트에서 다시한번 녹화를 한다면 그 내부의 어느부분이 문제인지 조금 더 특정할 수 있을 것이다.

profiler를 통해 얻을 수 있는 이점

결과적으로 profiler를 통해 얻을 수 있는 이점은 다음과 같다.

  1. 해당 페이지가 rendering되는 전체 시간의 파악이 가능하다. (User가 기다려야 하는 시간 확인가능)
  2. 해당 페이지에 불필요한 re-render로 인한 시간 및 성능 문제가 발생하는지 파악할 수 있다. (만일 있다면 React.memo 등으로 처리)
  3. 해당 페이지 내부의 컴포넌트 중 어느 컴포넌트의 문제로 인함인지 파악하여, 내부의 컴포넌트로 들어가 더 세부적인 원인을 파악할 수 있다.
