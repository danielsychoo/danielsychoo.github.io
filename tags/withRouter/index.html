withRouter와 history, location, match

처음에는 컴포넌트에 withRouter를 적용해야만 react-router자체가 적용된다고 생각했었음, 이후 Link 태그를 이용하지 않고 이동할때에 history가 필요함을 깨닫고, history를 이용하려면 withRouter를 적용시켜야 한다고 생각함.

이 과정에서 사수가 react-hook중 useHistory를 이용하면 된다고 하여 동일한 과정인데, useHistory를 이용해 조금 더 편하게 사용하는 것이라 생각했는데, 그게 포인트가 아니었음.

withRouter 또는 useHistory를 사용하는 경우는
route가 적용되지 않은 컴포넌트에서 history, location, match를 사용하는 경우임. 이를 반대로 말하면 route가 적용된 컴포넌트에서는 withRouter, useHistory를 사용하지 않아도 각 기능들이 props로 넘어온다는 것.

modal outsideClick 구현

modal을 구현하는 과정에서 modal외에 다른 부분을 클릭했을 때 modal이 닫히도록 하는 로직.

해당 모달 창(컴포넌트)에 useRef를 이용해 reference를 걸어놓고, component가 존재하는 상황에서 외부를 클릭하면 닫히도록 한다.
