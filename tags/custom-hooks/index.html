custom hook의 확장성

custom hooks를 공부하고 사용하는 과정에서 custom hook이란 결국 setState를 하는 로직을 custom하는 것이라고 생각했는데, 그 이상으로 생각해야 함.

custom hooks에는 꼭 [state, setState]만이 아니라 내가 필요로 하는 다른 것도 만들어 넣고 비구조화 할당을 통해 가져올 수 있음.

아래는 input에 대한 state관리와 reset을 할 수 있는 함수를 가진 custom hook 예시임.

이 외에도 추가적으로 더 필요한 부분들이 발생한다면 해당 기능에 대한 함수를 구현하여 return 후 사용하면 됨.

또한 기본적인 useState같은 hook들은 const [state, setState] = useState(initialState)와 같이 배열로 비구조화 할당을 했던 것과 달리 위 예시처럼 객체로 반환할 경우 객체로 비구조화 할당을 진행해야 하는데,

이때, 사용하는 변수명을 { 설정값: 사용할이름}의 방식으로 지정해줄 수 있음.

이외에 공부한 점

apollo-client에서 query의 variables

apollo-client에서 query 및 mutation을 날릴때 variables가 필요한 떄가 있음. 예를들어 list의 특정 요소를 Delete를 할 때에는 해당 요소의 id가 필요하기 때문에 variables로 id가 필요함.

이 경우 variables의 값으로 객체 {}가 들어감을 잊지 말아야 함.

또한 서버에서 id와 같은 값을 Int형으로 요청할경우 통신 후 json으로 인해 String으로 변환될 수 있다는 점을 인지하여 parseInt()를 씌워주는 것도 잊지 말자.

custom hooks 공부 (2021-04-07 수)

공부한 내용: useReducer, custom Hooks, scss

useReducer

1) reducer 함수

state와 action을 parameter로 받아와서 action에 해당하는 조치 후 nextState를 반환하는 함수

2) action

action이란 업데이트를 위한 정보를 의미

3) useReducer

일반적으로 useReducer를 사용하기위해 개발하는 순서

  1. 해당 component에서 사용할 initialState를 component외부 스코프에 선언
  2. reducer 함수의 틀만 만들어주고 component에 useReducer 사용
  3. state에서 필요한 값들을 destructuring해서 각 component에 전달
  4. action 및 reducer 함수 코드 작성

Custom Hooks

  • 반복되는 로직을 custom hooks으로 분리해 재사용 할 수 있도록 하려는 목적
  • 일반적으로 src/hooks에 모아놓음

따라서 custom hooks는

  1. Custom Hooks 안에서 useState, useEffect, useReducer, useCallback 등을 사용하여 원하는 기능을 구현한 후
  2. component에서 구현한 custom hooks를 사용해 원하는 값을 반환

하는 구조로 이루어진다. custom hooks를 만들고나면 결국 [currentValue, setCurrentValue] = customHook();의 형태가 되고, setCurrentValue가 custom되는 것이다.

SCSS

  • scss는 css의 pre-processor이다.
  • 따라서 scss는 typescript를 javascript로 compile하는 것과 같이 css로 compile하는 과정이 필요하다.
  • 그러나 react 에서는 compile의 과정이 필요하지 않다.
  • scss의 주요 기능들
    • 중첩(Nesting)
    • 상위 선택자 참조(Ampersand): &
    • 중첩 벗어나기: @at-root
    • 변수(variables): $
      • 이때, 변수의 유효범위는 Block scope에 따른다.
    • 믹스인(Mixin): Mixin 믹스인이름($parameter) {스타일;}
      • 설정 후 사용시: @include 믹스인이름(parameter)
    • 조건문(if)

결과적으로 css보다 다양한 기능을 js처럼 쓰고, compile을 통해 css로 변환하는 css의 superset.
