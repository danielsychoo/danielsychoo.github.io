GraphQL yoga кіөл¶Җ (2021-04-09 кёҲ)

GraphQL-yoga

GraphQLлЎң н•ҙкІ°н•  мҲҳ мһҲлҠ” л¬ём ңл“Ө

  1. Over-fetching: мҡ”мІӯм—җ лҢҖн•ҙ н•„мҡ”н•ң data мқҙмғҒмңјлЎң dataлҘј л°ӣм•„мҷҖ л№„нҡЁмңЁм Ғмқё мғҒнҷ©
  2. Under-fetching: н•„мҡ”н•ң м—¬лҹ¬ dataлҘј мң„н•ҙ м—¬лҹ¬лІҲ мҡ”мІӯн•ҙм•ј н•ҳлҠ” л№„нҡЁмңЁм Ғмқё мғҒнҷ©

GraphQLмқҖ л‘җ к°Җм§Җмқҳ нҠ№м§•м—җм„ң мқҙлҹ¬н•ң л¶Җл¶„л“Өмқ„ н•ҙкІ°н•  мҲҳ мһҲмқҢ.

  1. мҡ”мІӯмқ„ н•ҳлҠ” endpointк°Җ н•ҳлӮҳлЎң лҗҳм–ҙмһҲмқҢ. л”°лқјм„ң к°Ғк°Ғмқҳ dataлҘј мң„н•ҙ к°Ғк°Ғ мҡ”мІӯн•  н•„мҡ”к°Җ м—ҶмқҢ.
  2. frontм—җм„ң м§Ғм ‘ queryлҘј мһ‘м„ұн•ҳм—¬ н•„мҡ”н•ң м •ліҙл§Ң л°ӣм•„мҳ¬ мҲҳ мһҲмқҢ.

нҷҳкІҪм„Өм •

Schema

graphql dirмқ„ мғқм„ұн•ҳм—¬ н•ҙлӢ№ dir лӮҙл¶Җм—җ schema.graphql, resolvers.jsлҘј мһ‘м„ұ.

1) graphql/schema.graphql

  • queryм—җ лҢҖн•ң typeл“ұмқ„ м§Җм •
  • clientлӢЁм—җм„ң м–ҙл–Ө queryлҘј ліҙлӮҙм•ј н• м§Җ, л¬ҙм—Үмқҙ н•„мҲҳ(isRequired)мқём§Җ л“ұ нҷ•мқё к°ҖлҠҘ.

2) graphql/resolvers.js

  • schema.graphqlм—җм„ң defineн•ң queryм—җ лҢҖн•ң мӢӨм ң return dataл“Өмқҙ мһҲмқҢ.

3) more Useful example

мӢӨм ң dataмқҳ кІҪмҡ° мң„ мҳҲмӢңмҷҖ к°ҷмқҙ к°„лӢЁн•ҳкІҢ лҗҳм–ҙмһҲлҠ” кІҪмҡ°лҠ” м—Ҷмқ„ кІғмһ„. л”°лқјм„ң мӢӨм ңмҷҖ к°ҷмқҙ мҳҲмӢңлҘј л“Өм–ҙліҙл©ҙ м•„лһҳмҷҖ к°ҷмқҙ лҗЁ.

мқҙмҷҖ к°ҷмқҙ schemaлҘј мһ‘м„ұн•ҙлҶ“мңјл©ҙ clientм—җм„ңлҠ” queryлҘј нҶөн•ҙ personмқҳ name, age, genderлҘј м·ЁмӮ¬м„ нғқн•  мҲҳ мһҲмқҢ.

мқҙмІҳлҹј clientлӢЁм—җм„ң н•„мҡ”н•ң м •ліҙл§Ң лҪ‘м•„мҳ¬ мҲҳ мһҲлӢӨлҠ” мһҘм җмқҙ мһҲмңјлӮҳ, л°ҳлҢҖлЎң лӘЁл“  м •ліҙлҘј к°Җм ёмҷҖм•ј н•  кІҪмҡ°м—җлҠ” queryлҘј лӘЁл‘җ мһ‘м„ұн•ҙмЈјм–ҙм•ј н•ҳлҠ” лІҲкұ°лЎңмӣҖ м—ӯмӢң мһҲлӢӨ.

GraphQL - Playground

GraphQLлЎң ServerлҘј кө¬м¶•н•ң л’Ө yarn startлҘј нҶөн•ҙ localhostлҘј м—ҙл©ҙ Playgroundк°Җ м—°кІ°лҗЁ. мқҙлҘј нҶөн•ҙ queryмҷҖ mutationмқ„ н…ҢмҠӨнҠён•ҙліј мҲҳ мһҲмқҢ. Postmanкіј лҢҖмқ‘лҗҳлҠ” м„ңл№„мҠӨлЎң мғқк°Ғн•  мҲҳ мһҲмқҢ.

clientм—җм„ң schemaмқҳ typeмқ„ лӘЁлҘјкІҪмҡ°

м•„лһҳмқҳ queryлҘј лӮ л Ө нҷ•мқё к°ҖлҠҘ
(мқҙмҷёмқҳ л¶Җл¶„л“ӨлҸ„ кіөмӢқл¬ём„ң Introspectionм—җм„ң нҷ•мқёк°ҖлҠҘ)

DataмӨ‘ нҠ№м • к°’мқ„ м°ҫлҠ” query мғқм„ұ

DataмӨ‘м—җм„ң мӣҗн•ҳлҠ” к°’л§Ң м°ҫмңјл Өл©ҙ filter, mapкіј к°ҷмқҖ л°©мӢқмңјлЎң кө¬нҳ„н•  мҲҳ мһҲлӢӨ. м•„лһҳмқҳ мҳҲмӢңлҠ” idлҘј нҶөн•ҙ нҠ№м • dataлҘј м°ҫлҠ” л°©мӢқмқ„ кө¬нҳ„н•ҳлҠ” м Ҳм°ЁмқҙлӢӨ.

1) filterн•ЁмҲҳ м •мқҳ

2) кё°мЎҙ queryм—җ н•ҙлӢ№ query м¶”к°Җ

3) resolversм—җ filtering logic м„Өм •

personм—җ лҢҖн•ң queryлҘј м§„н–үн•ң нӣ„ console.log(...args)лҘј м°Қм–ҙліҙл©ҙ лӮҳмҳӨлҠ” м—¬лҹ¬ argsл“Ө мӨ‘ {id: <н•ҙлӢ№к°’>}мқҙ л“Өм–ҙмһҲмқҢ. л”°лқјм„ң { id }мҷҖ к°ҷмқҙ destructuringмңјлЎң лҪ‘м•„лӮё idлҘј мқҙмҡ©н•ҙ key-valueмҢҚмңјлЎң valueлҘј лҪ‘м•„лӮҙм–ҙ getById()мқҳ parameterм—җ л„ЈмқҢ.

mutation

mutaionмқҖ dataлҘј ліҖнҷ”(м¶”к°Җ, мӮӯм ң л“ұ)мӢңнӮ¬л•Ң мӮ¬мҡ©н•ҳлҠ” л©”мҶҢл“ңмқҙлӢӨ. mutaionмқ„ мӮ¬мҡ©н•  л•Ңм—җлҠ” queryмҷҖ л§Ҳм°¬к°Җм§ҖлЎң schemaмҷҖ resolversм—җ м •мқҳн•ңлӢӨ. л”°лқјм„ң м„ё лӢЁкі„мқҳ мҲңм„ңлЎң мғқк°Ғн•ҙліј мҲҳ мһҲлӢӨ.

  1. mutationм—җм„ң нҳём¶ңн•  н•ЁмҲҳлҘј л§Ңл“ лӢӨ.
  2. schemaм—җ Mutationмқ„ м •мқҳн•ңлӢӨ.
  3. resolversм—җ Mutationмқ„ мӮ¬мҡ©н•ңлӢӨ.

мқҙнӣ„ мҝјлҰ¬л¬ёмқ„ лӮ л Ө мӮ¬мҡ©н•ҳлҠ” л°©лІ•мқҖ м•„лһҳмҷҖ к°ҷлӢӨ.
